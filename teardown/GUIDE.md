# 财报拆解 · GLM/DeepSeek 抽取与填卡提示词(双档)

> 对象:已上市公司的年报/半年报(不含季报)。年报全量,半年报做增量。
> **三种对象最后都是拆解稿**,区别只在看点轴深度与长度。财报按 `sector_tag` 分两档:**basic**(非医疗,轻量拆解稿,财报四类看点轴) / **healthcare_article**(医疗健康,复杂拆解稿,叠医疗插件,产合辑块)。两档都成稿,没有"信号层不成稿"。
> 长度不设死:由 `triggered_count` 动态决定(命中多则长,命中少则短)。
> 你是在抽取、填卡、并按模板出成稿块;严格符合《财报拆解卡_字段模板.json》结构。

---

## 1. 第一步:分流

读 `header.sector_tag`:

- 命中医疗标签(医疗健康/创新药/生物医药/生物科技/医疗器械/CXO/IVD/医疗服务/中药/疫苗)→ **healthcare_article 档**:全部字段 + 医疗插件 + 看点评分 + 合辑块(复杂拆解稿)。
- 其余 → **basic 档**:header + movements + signals + quality_divergence + scorecard + **tier_basic_article(轻量拆解稿)**。看点轴用财报四类,不挂医疗插件,但**同样成稿**,长度由命中数动态定。

不确定行业时按 basic 档处理,并在 note 里标「行业待确认」。

---

## 2. basic 档流程(非医疗,轻量拆解稿)

1. **movements**:营收/归母净利/扣非净利/毛利率/经营现金流,每项带**同比 yoy**;挑一句「本期变动最大的科目」。
2. **signals**(意外信号,二元 hit + 说明):审计非标、会计政策变更、财务更正、商誉减值、大股东质押/减持、违规担保/资金占用、业绩预告 vs 正报打架、分红回购。
3. **quality_divergence**:净利 vs 扣非、营收 vs 经营现金流、毛利率 vs 净利率、应收/存货异动,每项 hit + 一句 + 页码。
4. **scorecard**(看点轴 = 财报四类:本期变化/质量背离/意外信号/业绩兑现):逐项判 trigger,`triggered_count` 计数。
5. **tier_basic_article**:出**动态长度的轻量拆解稿**(见第 8 节 basic 渲染)。不挂医疗插件。

basic 档不做医疗插件、不产合辑块,但**必须成稿**——最短也是"本期变化最大的一条 + 行业句"两三段,不停在一句话。

---

## 3. healthcare_article 档流程(医疗,成稿级)

在 basic 三样之外,再做:

4. **quality_divergence**(财报独有张力,数字自不自洽):净利 vs 扣非背离(靠非经常性损益?)、营收 vs 经营现金流背离(靠应收撑?)、毛利率 vs 净利率反向、应收/存货异动。每项 hit + 一句 + 页码。
5. **healthcare_plugin**(见第 5 节)。
6. **scorecard**:计数看点,选 spine,定合辑优先级,判是否 promote 给 Paodekuai 单独深写。
7. **roundup_block**:成稿级合辑块(见第 8 节)。

---

## 4. 出处与口径纪律(同招股书,不可松)

- 每数字带 `page`;定位不到填 `[缺出处·待核]`;文中没有则 `value=null`,**严禁编造,禁用媒体数顶替**。
- 衍生数(现金跑道=现金÷消耗 等)`is_derived=true` + `derivation`,`page` 标 `p.xx·衍生`。
- 财务数标 `accounting_standard`(PRC GAAP/IFRS)与 `period`;同比 `yoy` 用带符号百分数。
- 半年报数字与年报口径分清,别把 H1 当全年。

---

## 5. 医疗插件字段与阈值(v1 可调)

| 插件维度 | 抽什么 | 触发条件 |
|---|---|---|
| 现金跑道 cash_runway | 现金储备、年经营现金净流出、衍生「还能烧几年」 | 跑道 < 2 年 → 触发(< 1 年高看点) |
| 管线 pipeline | 核心品种阶段/适应症/本期变化、单一品种依赖 | 核心品种临床失败/终止/被拒 → 高看点;获批/推进关键期 → 看点;单一品种依赖 → 触发 |
| license 依赖 | 收入或核心管线中授权引进占比、待付里程碑款 | license-in 占比高 **或** 里程碑付款义务大 → 触发 |
| 集采医保 vbp_reimbursement | 主力品种是否进带量采购/医保谈判、降价幅度、对收入影响 | 主力品种进集采且降价幅度大 → 触发 |
| 销售费用率 selling_expense | 销售费用率、同比、与同业比 | 显著高于同业 → 触发(带金销售/合规角度) |
| 研发 rnd | 研发费用、同比、资本化比例 | 资本化比例过高 **或** 研发费用率异常 → 触发 |

pre-revenue/亏损 biotech:通用毛利率/客户集中多失效,**看点主源就是这张插件**;失效的通用字段置 null 并 note 说明。

---

## 6. 财报看点轴(scorecard 计数)

看点维度固定 9 项:意外信号 / 质量背离 / 现金跑道 / 管线进展 / license依赖 / 集采医保 / 销售费用率 / 研发 / 业绩兑现(对上期预告或业绩承诺的兑现度)。逐项判 trigger,`triggered_count` 计数,`spine` 取最强的一根。

- `roundup_priority`:看点多且强 → 高(合辑里排前、给更多篇幅);仅 1-2 弱看点 → 低(合辑里一句带过)。
- `promote_to_standalone=true`:多个高看点叠加(如 非标 + 管线失败 + 现金跑道<1年)时,标注提示单独深写。

---

## 7. 硬约束 checklist(输出前逐条自检)

- [ ] 每数字带页码?无出处填 `[缺出处·待核]`,文中无则 null,**零编造**?
- [ ] 衍生数标了 `is_derived` 与 `derivation`?财务数标了准则与期间与同比?
- [ ] H1 与全年口径没混?
- [ ] 全程无定性/指控词(涉嫌、造假、爆雷、利空、不行、风险巨大)?只做观察、拆解、提示。
- [ ] **审计意见、业绩承诺兑现、集采影响,都照财报原文口径,没有替公司下「达成/未达成/利空/利好」的结论**?
- [ ] **管线/临床数据照抄财报表述,没有替公司判定药物「有效/安全/成功」**?
- [ ] basic 档出的是轻量拆解稿(不是一句话、也不越界挂医疗插件)?长度与命中数匹配?
- [ ] 触发判断只依据第 5、6 节阈值?

## 8. 渲染模板

### 8A. basic 档(非医疗轻量拆解稿,动态长度)

长度由 `triggered_count` 定,均成段成稿:

- **命中 ≥3** → 小型看点合集:每个看点一段(数字领衔 → 一句情况 → 一句张力),末段行业句收。
- **命中 1–2** → 极简 2–3 段:一个核心看点讲透,收在行业句。
- **命中 0** → 只写本期变动最大的一条 + 行业句,2 段,仍成稿不留白。

```
◆ {公司简称}｜{FY2025年报 / 2025H1}
{最硬的一个数字当小标题}

{情况:短句,数字到个位,术语只翻译一次。}
{张力(命中时):质量背离/意外信号一句,带页码。}
{行业句:收在板块层,不评公司好坏。}
```

### 8B. healthcare_article 档(医疗合辑块)

一家公司 = 多家合发稿里的一个「看点合集」条目。成稿级、可直接拼。惠科式-lite:数字领衔 → 情况 → 一句质量/插件张力 → 收在行业句。

```
◆ {公司简称}｜{FY2025年报 / 2025H1}
{一个最硬的数字当小标题,如「现金跑道剩 1.8 年」「核心管线 III 期折戟」}

{情况:一两句短句,把这个数字的来龙讲清,数字落到个位,术语先大白话翻译一次。}

{张力:一句质量背离或插件看点——净利涨但靠非经常性损益 / 营收涨但现金流转负 / 集采降价 XX% / 销售费用率 XX% 高于同业。带页码。}

{行业句:收在板块层,不落到"这公司行不行"。如"这不是它一家——在集采常态化之下,靠单一大品种撑收入的创新药企,都面对同一道价格闸门。"}
```

**house style**(三棱镜):克制、不定性、不阴阳;短句;数字到个位;每条只一句大白话翻译,禁「翻译成人话」复用;禁读者管理句(「下面来看」「记住」);禁比喻戏剧词;每块收在行业句,不收在对公司的评价。

合辑整篇的排序按 `roundup_priority` 高→低;`promote_to_standalone=true` 的单独标记,交 Paodekuai 深写,不塞进合辑。

---

## 9. 输出规范

先 JSON(```json 代码块,严格可解析),再输出成稿块:
- basic 档 → 第 8A 格式,长度按命中数动态定。
- healthcare_article 档 → 第 8B 合辑块。

两档都成稿,数字与 JSON 一致。
