# 怎么把它变成"每天打开一个网址就能看"(全程点鼠标,不碰终端)

目标:设置一次(约 15 分钟),之后每周一到五它自动扫四所,把结果更新到一个网页。
你日常"用"它 = 打开收藏的那个网址。以下每一步都是在网页上点,不需要终端、不需要装 Python。

---

## 一、注册一个 GitHub 账号(如果没有)
去 github.com,点 Sign up,用邮箱注册。免费。

## 二、新建一个仓库(repository)
1. 登录后,右上角 "+" → New repository。
2. 名字随便起,比如 `ipo-monitor`。
3. 选 **Private(私有)** 也行、Public 也行(想让网页能公开访问就选 Public)。
4. 点 Create repository。

## 三、把 ipo_monitor 文件夹传上去
1. 在新仓库页面,点 "uploading an existing file"(或 Add file → Upload files)。
2. 把我给你的 `ipo_monitor` 文件夹里的**所有内容**拖进去(直接拖文件夹进浏览器即可)。
   注意:要连 `.github` 这个文件夹一起传(它是自动运行的配置)。如果拖拽没带上它,
   见文末"补传 .github"。
3. 拖完后,页面下方点绿色的 Commit changes。

## 四、打开"自动运行"开关
1. 仓库页面上方点 **Actions** 标签。
2. 如果提示 "Workflows aren't being run",点 **I understand… enable them**。
3. 左侧会看到 "IPO每日监控",点它 → 右边 **Run workflow** → 绿色 Run workflow。
   (这会立刻先跑一次,几分钟后仓库里会出现 dashboard.html / index.html。)

## 五、打开"网页"开关(GitHub Pages)
1. 仓库页面上方 **Settings** → 左侧 **Pages**。
2. Source 选 **Deploy from a branch**;Branch 选 **main**,文件夹选 **/ (root)**,点 Save。
3. 等一两分钟,这一页顶部会出现你的网址,形如
   `https://你的用户名.github.io/ipo-monitor/`
4. **把这个网址收藏。以后每天点开它,就是你的监控看板。**

设置完成。之后每周一到五早上它自动跑、自动更新这个网页,你什么都不用做。

---

## 日常怎么用
- 打开收藏的网址。
- 顶部看今天新增几家、几家 ★可选题。
- ★可选题 = 已过会 / 已发聆讯后资料集及以后 = 值得你考虑写的。
- 点公司下面的"招股书 / 聆讯后资料集"链接,直接打开原始 PDF。

## 想手动立刻刷新一次
Actions → IPO每日监控 → Run workflow。

## 补传 .github(万一没传上)
Add file → Create new file,文件名处输入 `.github/workflows/daily.yml`(斜杠会自动建文件夹),
把 daily.yml 的内容粘进去,Commit。

## 不想自己弄这些?
这个包代码和测试都齐了,部署是很小的活。你可以把整个 `ipo_monitor` 文件夹连同这份指南
丢给任何一个会点技术的人(或找个freelancer),他照着 GitHub 方案或放自己服务器,半小时能搞定。


## 开启"拆解档案"自动生成(可选,需一次配置)
过会公司想自动生成"说人话"拆解档案的话,需要一个 Anthropic API key:
1. 到 console.anthropic.com 注册并创建一个 API key(以 sk-ant- 开头)。
2. 回到你的 GitHub 仓库 → Settings → Secrets and variables → Actions → New repository secret。
3. Name 填 `ANTHROPIC_API_KEY`,Value 粘贴你的 key,保存。
之后每次自动运行,过会公司会在 data/dossiers/ 下各生成一篇档案(机器底稿,数字带页码,
闸门自动拦截无出处数字与定性措辞)。不配置这个 key,监控和网页照常运行,只是不建档。
费用参考:只给过会公司建档,一周几家,每篇几块钱人民币量级。
